API — Pipeline Integration

Use the POST /api/v1/generate-report endpoint to generate reports directly from your CI/CD pipeline.

Pipe semgrep output directly

Run semgrep with JSON output and pipe it straight to the API. The report PDF is saved locally.

semgrep --json | curl -X POST https://sast-report-generator.onrender.com/api/v1/generate-report \
-H "Content-Type: application/json" -d @- -o sast_report.pdf

With repository name

Pass a repo name via query parameter to label the report.

semgrep --json | curl -X POST "https://sast-report-generator.onrender.com/api/v1/generate-report?repo_name=my-app" \
-H "Content-Type: application/json" -d @- -o sast_report.pdf

Upload a saved JSON file

If you already have a semgrep results file, upload it with the file field.

curl -X POST https://sast-report-generator.onrender.com/api/v1/generate-report \
-F "file=@results.json" -F "repo_name=my-app" -o sast_report.pdf

Request Parameters

Parameter Location Required Description
file multipart form required* Semgrep JSON results file
repo_name form / query optional Name of the repository for the report header

* For JSON body requests, the entire semgrep JSON is sent as the request body instead of the file field.

Response

Success: 200 OK — returns the PDF with Content-Type: application/pdf.
Error: 400 / 500 — returns a JSON object with an error field.
