API — Pipeline Integration

Use the POST /api/v1/generate-report endpoint to generate reports directly from your CI/CD pipeline. Auto-detects Semgrep, SARIF (CodeQL, Trivy, etc.), and Snyk formats.

Semgrep

Pipe semgrep JSON output directly to the API.

semgrep --json | curl -X POST https://sast-report-generator.onrender.com/api/v1/generate-report \
-H "Content-Type: application/json" -d @- -o sast_report.pdf

SARIF (CodeQL, Trivy, etc.)

Pipe any tool that outputs SARIF format.

codeql database analyze --format=sarif-latest | curl -X POST \
https://sast-report-generator.onrender.com/api/v1/generate-report \
-H "Content-Type: application/json" -d @- -o sast_report.pdf

Snyk

Pipe Snyk test JSON output.

snyk test --json | curl -X POST https://sast-report-generator.onrender.com/api/v1/generate-report \
-H "Content-Type: application/json" -d @- -o sast_report.pdf

With repository name

Pass a repo name via query parameter to label the report.

semgrep --json | curl -X POST "https://sast-report-generator.onrender.com/api/v1/generate-report?repo_name=my-app" \
-H "Content-Type: application/json" -d @- -o sast_report.pdf

Multi-tool merge (JSON array)

Send an array of scan results to merge into one report.

# Pipe two tools side by side: (semgrep --json; snyk test --json) | jq -s '.' | curl -X POST \ https://sast-report-generator.onrender.com/api/v1/generate-report \ -H "Content-Type: application/json" -d @- -o sast_report.pdf

Single file

Upload a saved JSON results file.

curl -X POST https://sast-report-generator.onrender.com/api/v1/generate-report \
-F "file=@results.json" -F "repo_name=my-app" -o sast_report.pdf

Multi-file merge

Upload multiple files — use a separate -F "file=@..." for each.

curl -X POST https://sast-report-generator.onrender.com/api/v1/generate-report \
-F "file=@semgrep.json" -F "file=@snyk.json" -F "repo_name=my-app" -o sast_report.pdf

Request Parameters

Parameter Location Required Description
file multipart form required* Security scan results (Semgrep, SARIF, Snyk). Use multiple file fields to merge tools.
repo_name form / query optional Name of the repository for the report header

* For JSON body requests, the entire JSON output is sent as the request body instead of the file field. Tool format is auto-detected (Semgrep, SARIF, Snyk).

Response

Success: 200 OK — returns the PDF with Content-Type: application/pdf.
Error: 400 / 500 — returns a JSON object with an error field.
